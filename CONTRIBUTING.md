# Contributing to Envman

First off, thank you for considering contributing to Envman! It's people like you that make Envman such a great tool for the community.

## 1. Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](https://github.com/fronik/envman/issues) first to see if someone else has already created it. 

## 2. Setting up your environment

1. Fork the repo and clone it locally
2. Run `npm install`
3. Run tests using `npm test`

## 3. Making Changes

- Ensure your code follows the existing style.
- We use Jest for testing. If you add a new feature, please add a corresponding test in the `test/` directory.
- `npm run test` must pass before we can accept a PR.

## 4. Submitting a Pull Request

1. Push your changes to your fork.
2. Submit a pull request to the `main` branch.
3. Provide a clear and descriptive title and explain the changes you've made.

Thanks for your help!
